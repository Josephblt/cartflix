# Quips

File: runtime `quips.json`

Quips store the short personality lines Cartflix shows around the app.

They are runtime data, not business records. They do not affect grocery
operations, catalog matching, history, authentication, or Carty's operating
rules.

## File Shape

```json
{
  "type": "cartflixQuips",
  "version": 1,
  "openingQuips": [
    {
      "quipId": "018f6a3d-7b8e-7a11-9f50-2c2c2edc0201",
      "text": "The first rule of Cart Club is: You do not talk about Cart Club."
    }
  ],
  "cartyGreetingQuips": [
    {
      "quipId": "018f6a3d-7b8e-7a11-9f50-2c2c2edc0202",
      "text": "What does the fridge deny needing?"
    }
  ]
}
```

## Quips

```json
{
  "type": "cartflixQuips",
  "version": 1,
  "openingQuips": [],
  "cartyGreetingQuips": []
}
```

Fields:

- `type`: file type marker. Initial value is `cartflixQuips`.
- `version`: schema version. Initial value is `1`.
- `openingQuips`: short comic lines shown by Cartflix during opening or login
  surfaces.
- `cartyGreetingQuips`: short greeting prompts shown when the Carty chat surface
  opens.

## Quip Model

A quip is a short display line with a stable opaque UUID.

```json
{
  "quipId": "018f6a3d-7b8e-7a11-9f50-2c2c2edc0201",
  "text": "The first rule of Cart Club is: You do not talk about Cart Club."
}
```

Fields:

- `quipId`: stable opaque quip UUID.
- `text`: display text.

Rules:

- `quipId` values are generated by Cartflix.
- `text` must be non-empty after trimming whitespace.
- `text` must not contain newlines.
- `text` should be short enough to fit a compact mobile surface.
- duplicate quip text inside the same array should be avoided.
- order is meaningful for review and index-based replacement, but display order
  may be randomized by the app

## Ownership Boundary

`openingQuips` belong to Cartflix's UI voice.

`cartyGreetingQuips` are shown near Carty, but they are still app display data.
They are not Carty instructions, agent policy, or conversational memory.

If Carty needs durable operating rules, those belong under `agent/`, not in
runtime `quips.json`.

## Selection Behavior

The app may select a quip at random or rotate through the configured list. The
high-level `getQuip` app operation returns a random quip for the requested
collection.

Recommended baseline:

- avoid showing the same quip twice in a row when more than one option exists
- fall back to a small built-in default if the file is missing, invalid, or the
  relevant array is empty
- do not write selection history into `quips.json`

## Operation Behavior

Low-level quip operations are deliberately small:

- add a quip
- get a quip by zero-based index
- remove a quip

There is no low-level edit operation for quips. If a quip should change, the
high-level app operation replaces the quip at a specific index by reading that
index, removing the current quip, and inserting the new quip at the same index.

## Deliberate Omissions

Quips do not have weights, tags, authors, locales, or enabled flags in the
initial model.

Those fields can be added later if the list grows enough to need editorial
management. For now, the file should stay easy to read and edit by hand.
